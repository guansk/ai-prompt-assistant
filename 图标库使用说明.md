# 图标库使用说明

## 使用的图标库

本项目使用 **Lucide Icons** - 一个现代、轻量、开源的图标库。

- 官网：https://lucide.dev/
- GitHub：https://github.com/lucide-icons/lucide
- 图标数量：1000+ 个
- 文件大小：~60KB (CDN)
- 许可证：ISC License (完全免费商用)

## 为什么选择 Lucide Icons？

### 1. 轻量高效
- 比 Font Awesome 小 80%+
- 按需加载，性能优秀
- 支持 Tree-shaking

### 2. 设计优秀
- 24x24 网格设计
- 2px 描边宽度
- 圆角风格统一
- 视觉一致性强

### 3. 易于使用
- 简单的 HTML 标签
- 自动替换为 SVG
- 支持 CSS 样式控制

### 4. Chrome 插件友好
- 支持 CDN 引入
- 无需构建工具
- 兼容 CSP 策略

## 集成方式

### 1. 下载到本地（推荐）

Chrome 扩展不允许使用外部 CDN，需要将图标库下载到本地：

```bash
# 创建 libs 目录
mkdir libs

# 下载 Lucide Icons
curl -o libs/lucide.min.js https://unpkg.com/lucide@latest/dist/umd/lucide.min.js
```

在 HTML 的 `<head>` 中引入：

```html
<script src="libs/lucide.min.js"></script>
```

### 2. 使用 CDN（仅用于测试页面）

在非扩展页面（如 test-icons.html）中可以使用 CDN：

```html
<script src="https://unpkg.com/lucide@latest"></script>
```

### 2. 使用图标

在 HTML 中使用 `data-lucide` 属性：

```html
<i data-lucide="search"></i>
<i data-lucide="settings"></i>
<i data-lucide="refresh-cw"></i>
```

### 3. 初始化图标

在页面加载完成后调用：

```javascript
lucide.createIcons();
```

### 4. 动态内容

如果动态添加了包含图标的 HTML，需要再次调用：

```javascript
// 添加新内容后
document.getElementById('container').innerHTML = '<i data-lucide="star"></i>';
// 重新初始化
lucide.createIcons();
```

## CSS 样式控制

### 图标大小

```css
.btn-icon svg {
  width: 18px;
  height: 18px;
}

.empty-icon {
  width: 48px;
  height: 48px;
}
```

### 图标颜色

```css
.btn-icon {
  color: #6b7280; /* 图标会继承这个颜色 */

.empty-icon.warning {
  color: #f59e0b;
}

.empty-icon.error {
  color: #ef4444;
}
```

### 图标动画

```css
.btn-refresh.loading svg {
  animation: rotate 1s linear infinite;
}

@keyframes rotate {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

## 性能优化建议

### 1. 避免频繁初始化

```javascript
// ❌ 不好的做法
setInterval(() => {
  lucide.createIcons();
}, 100);

// ✅ 好的做法
function addNewContent() {
  container.innerHTML = newContent;
  lucide.createIcons(); // 只在需要时调用
}
```

### 2. 使用事件委托

```javascript
// ❌ 不好的做法
document.querySelectorAll('.btn').forEach(btn => {
  btn.addEventListener('click', handler);
});

// ✅ 好的做法
document.addEventListener('click', (e) => {
  if (e.target.closest('.btn')) {
    handler(e);
  }
});
```

### 3. 缓存图标元素

```javascript
// 如果图标不会改变，可以缓存 DOM 引用
const refreshIcon = document.querySelector('[data-lucide="refresh-cw"]');
```

## 离线使用方案（已实现）

本项目已将 Lucide Icons 下载到本地 `libs/` 目录：

```
your-extension/
├── libs/
│   └── lucide.min.js  ← 已下载
├── popup.html
└── manifest.json
```

这样做的好处：
- ✅ 符合 Chrome 扩展 CSP 策略
- ✅ 完全离线可用
- ✅ 加载速度更快
- ✅ 不依赖外部服务

如需更新图标库版本：

```bash
# 下载最新版本
curl -o libs/lucide.min.js https://unpkg.com/lucide@latest/dist/umd/lucide.min.js

# 或下载指定版本
curl -o libs/lucide.min.js https://unpkg.com/lucide@0.263.1/dist/umd/lucide.min.js
```

## 最佳实践

### 1. 语义化使用

```html
<!-- ✅ 好的做法 -->
<button class="btn-refresh" title="刷新">
  <i data-lucide="refresh-cw"></i>
</button>

<!-- ❌ 不好的做法 -->
<div onclick="refresh()">
  <i data-lucide="refresh-cw"></i>
</div>
```

### 2. 提供文字说明

```html
<!-- 对于重要操作，提供文字 -->
<button class="btn-primary">
  <i data-lucide="send"></i>
  <span>生成并填入</span>
</button>

<!-- 对于图标按钮，提供 title -->
<button class="btn-icon" title="设置">
  <i data-lucide="settings"></i>
</button>
```

### 3. 保持一致性

- 同类操作使用相同图标
- 图标大小保持统一
- 颜色使用符合语义

### 4. 无障碍访问

```html
<button aria-label="刷新数据">
  <i data-lucide="refresh-cw" aria-hidden="true"></i>
</button>
```

## 故障排查

### 图标不显示

1. 检查文件是否存在：`libs/lucide.min.js`
2. 检查是否调用了 `lucide.createIcons()`
3. 检查图标名称是否正确（访问 https://lucide.dev/icons）
4. 打开浏览器开发工具查看控制台错误

### 图标样式异常

1. 检查 CSS 是否正确设置了 `width` 和 `height`
2. 检查是否有其他样式冲突
3. 使用浏览器开发工具检查生成的 SVG

### 动态内容图标不显示

在添加新内容后，记得调用：

```javascript
lucide.createIcons();
```
