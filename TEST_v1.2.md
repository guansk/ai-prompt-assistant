# v1.2.0 功能测试清单

## 测试环境准备

1. 确保已加载插件到 Chrome
2. 准备两种格式的飞书多维表链接：
   - 完整链接：`https://xxx.feishu.cn/base/xxxxx?table=xxxxx`
   - 简化链接：`https://xxx.feishu.cn/base/xxxxx`

## 功能一：简化链接支持

### 行为说明

当使用简化链接（不带 `?table=xxx` 参数）时：

1. **保存配置时**：
   - 自动尝试获取第一个表格的 ID
   - 成功获取后永久保存到 storage
   - 如果获取失败，仍然保存配置，使用时再获取

2. **使用时（popup）**：
   - 优先使用已保存的 tableId
   - 只有当 tableId 不存在时才请求 API 获取
   - 获取后永久保存，下次直接使用

3. **测试连接时**：
   - 如果没有 tableId，会获取并保存
   - 显示提示信息告知用户

### 测试步骤

1. 打开插件设置页面
2. 填写 App ID 和 App Secret
3. 在"多维表链接"字段粘贴简化链接（不带 `?table=xxx` 参数）
4. 点击"测试连接"

### 预期结果

- ✅ 连接测试成功
- ✅ 显示消息："连接测试成功！使用 飞书/Lark API（自动使用第一个表格: tblXXXXXX）"
- ✅ 配置自动保存 tableId
- ✅ 再次测试连接时，不再显示"自动使用第一个表格"提示（因为已保存）

### 性能验证

1. 第一次保存配置（简化链接）：
   - 会发起 API 请求获取 tableId
   - 保存成功后，tableId 已存储

2. 第二次打开 popup：
   - 直接使用已保存的 tableId
   - 不会再次请求获取 tableId
   - 可以通过浏览器开发者工具的 Network 面板验证

3. 验证方法：
   ```javascript
   // 在浏览器控制台执行
   chrome.storage.local.get(['tableId'], (result) => {
     console.log('已保存的 tableId:', result.tableId);
   });
   ```

## 功能二：配置导出

### 导出时的智能处理

导出配置时会智能处理 tableId：

1. **如果已有 tableId**：
   - 直接包含在导出文件中
   - 无需额外请求

2. **如果没有 tableId**：
   - 自动尝试获取第一个表格的 ID
   - 获取成功：包含在导出文件中，并保存到本地
   - 获取失败：导出 `null`，提示用户

### 测试步骤

#### 场景 1：已有 tableId 的导出

1. 确保已保存有效配置（包含 tableId）
2. 在设置页面点击"导出配置"按钮
3. 检查下载的 JSON 文件

#### 场景 2：没有 tableId 的导出

1. 清除 tableId：
   ```javascript
   chrome.storage.local.remove('tableId');
   ```
2. 点击"导出配置"按钮
3. 观察提示信息和下载的文件

### 预期结果

**场景 1（已有 tableId）：**
- ✅ 自动下载 JSON 文件
- ✅ 文件名格式：`team-prompt-config-YYYY-MM-DD.json`
- ✅ 文件包含完整的 tableId
- ✅ 显示消息："✓ 配置已导出（包含表格 ID: tblXXXXXX）"

**场景 2（没有 tableId）：**
- ✅ 显示"正在获取表格信息..."
- ✅ 自动获取第一个表格的 ID
- ✅ 获取成功后包含在导出文件中
- ✅ 显示消息："✓ 配置已导出（包含表格 ID: tblXXXXXX）"
- ✅ tableId 同时保存到本地 storage

**场景 2（获取失败）：**
- ✅ 仍然导出配置文件
- ✅ tableId 为 null
- ✅ 显示消息："✓ 配置已导出（未包含表格 ID，导入后会自动获取）"

### JSON 文件验证

检查文件内容是否包含：
```json
{
  "version": "1.0",
  "exportTime": "2026-03-10T...",
  "config": {
    "appId": "cli_...",
    "appSecret": "...",
    "bitableUrl": "https://...",
    "appToken": "...",
    "tableId": "..." // 或 null
  }
}
```
