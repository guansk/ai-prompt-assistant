# v1.2.0 功能测试清单

## 测试环境准备

1. 确保已加载插件到 Chrome
2. 准备两种格式的飞书多维表链接：
   - 完整链接：`https://xxx.feishu.cn/base/xxxxx?table=xxxxx`
   - 简化链接：`https://xxx.feishu.cn/base/xxxxx`

## 功能一：简化链接支持

### 测试步骤

1. 打开插件设置页面
2. 填写 App ID 和 App Secret
3. 在"多维表链接"字段粘贴简化链接（不带 `?table=xxx` 参数）
4. 点击"测试连接"

### 预期结果

- ✅ 连接测试成功
- ✅ 显示消息："连接测试成功！使用 飞书/Lark API（自动使用第一个表格: tblXXXXXX）"
- ✅ 配置自动保存 tableId

### 测试用例

#### 用例 1：完整链接（向后兼容）
- 输入：`https://xxx.feishu.cn/base/PMgZbXR4davz22ssdmQcxDfQnjf?table=tblXXXXXX`
- 预期：正常解析 appToken 和 tableId

#### 用例 2：简化链接（新功能）
- 输入：`https://xxx.feishu.cn/base/PMgZbXR4davz22ssdmQcxDfQnjf`
- 预期：解析 appToken，自动获取第一个表格的 tableId

#### 用例 3：国际版链接
- 输入：`https://xxx.larksuite.com/base/PMgZbXR4davz22ssdmQcxDfQnjf`
- 预期：正常解析

## 功能二：配置导出

### 测试步骤

1. 确保已保存有效配置
2. 在设置页面点击"导出配置"按钮
3. 检查下载的 JSON 文件

### 预期结果

- ✅ 自动下载 JSON 文件
- ✅ 文件名格式：`team-prompt-config-YYYY-MM-DD.json`
- ✅ 文件内容包含所有配置信息
- ✅ 显示成功消息："✓ 配置已导出"

### JSON 文件验证

检查文件内容是否包含：
```json
{
  "version": "1.0",
  "exportTime": "2026-03-10T...",
  "config": {
    "appId": "cli_...",
    "appSecret": "...",
    "bitableUrl": "https://...",
    "appToken": "...",
    "tableId": "..." // 或 null
  }
}
```

## 功能三：配置导入

### 测试步骤

1. 准备一个有效的配置 JSON 文件（可以使用刚导出的）
2. 清空当前配置（或使用新的浏览器 profile）
3. 在设置页面点击"导入配置"按钮
4. 选择 JSON 文件

### 预期结果

- ✅ 表单自动填充配置信息
- ✅ 配置自动保存到 storage
- ✅ 显示成功消息："✓ 配置已导入并保存"

### 错误处理测试

#### 用例 1：无效的 JSON 格式
- 操作：导入格式错误的 JSON 文件
- 预期：显示错误消息 "JSON 格式错误，请检查文件内容"

#### 用例 2：缺少必要字段
- 操作：导入缺少 appId 的 JSON 文件
- 预期：显示错误消息 "配置文件格式不正确，缺少必要字段"
