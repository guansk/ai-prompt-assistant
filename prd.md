# 产品需求文档 (PRD)：Team Prompt Manager 浏览器插件

## 1. 产品概述

### 1.1 项目背景

在日常使用大语言模型（如 ChatGPT、Claude）时，团队成员往往会沉淀出大量优秀的 Prompt。但目前这些 Prompt 散落在个人的备忘录或文档中，团队内部无法有效共享，且每次使用都需要经历“查找 -> 复制 -> 切换页面 -> 粘贴”的繁琐步骤，协作效率低下。

### 1.2 产品目标

开发一款轻量级的 Chrome 浏览器插件，以**飞书多维表**作为底层数据管理后台，实现 Prompt 的团队级共享、统一维护，以及在 AI 对话界面的一键快捷填充。

### 1.3 用户角色

* **普通使用者（团队成员）：** 在 AI 工具页面快速查找并调用团队沉淀的 Prompt。
* **内容维护者（全员或指定人员）：** 在飞书多维表中新增、修改、删除或分类 Prompt，对所有人实时生效。

---

## 2. 核心价值与应用场景

* **Low-Code 后台免开发：** 利用飞书天然的权限管理、历史回滚和多端同步能力，省去传统管理后台的开发与维护成本。
* **沉浸式调用：** 用户停留在 ChatGPT/Claude 页面即可完成 Prompt 的检索和输入，无需打断当前工作流。
* **业务场景：** 客服团队共享标准化话术、研发团队共享代码审查/重构模板、设计/文案团队共享特定风格的生图/写作提示词。

---

## 3. 产品架构

* **前端（交互层）：** Chrome 浏览器插件（基于 Manifest V3 标准）。
* **后端（数据层）：** 飞书开放平台 API + 飞书多维表 (Bitable)。
* **数据流向：** 插件发起请求 -> 飞书 API 鉴权 -> 拉取多维表数据 -> 缓存在浏览器本地 -> 用户点击 -> 注入目标网页 DOM。

---

## 4. 功能需求详细说明

### 4.1 数据管理端（飞书多维表）

本产品无需开发管理后台，但需要定义多维表的数据结构标准（Schema）。

| 字段名称 | 字段类型 | 必填 | 业务说明 |
| --- | --- | --- | --- |
| **Title** | 单行文本 | 是 | Prompt 的标题或短语（如：Python 代码重构） |
| **Content** | 多行文本 | 是 | 实际需要输入给 AI 的完整提示词内容 |
| **Category** | 单选标签 | 否 | 用于插件端的筛选（如：研发、设计、产品） |
| **Status** | 单选标签 | 否 | 控制是否展示（如：启用/停用），便于下线废弃的提示词 |

### 4.2 插件配置模块 (Options)

**功能描述：** 提供一个独立页面，供用户配置飞书 API 凭证及多维表数据源。
**功能清单：**

1. **表单输入：** 包含 `App ID`、`App Secret`、`App Token` (多维表标识)、`Table ID` (数据表标识) 四个必填项。
2. **本地保存：** 点击“保存配置”后，将数据安全存储在浏览器本地（`chrome.storage.local` 或 `sync`）。
3. **状态反馈：** 保存成功后给予明确的 UI 提示（如绿色文字“保存成功”）。

### 4.3 插件核心交互端 (Popup)

**功能描述：** 用户点击浏览器右上角插件图标后弹出的核心操作界面。
**功能清单：**

1. **数据加载与缓存：** * 打开弹窗时，优先展示本地缓存的 Prompt 列表（提升开启速度）。
* 后台异步请求飞书 API 获取最新数据，若有更新则静默刷新列表。
* 加载状态展示：API 请求期间展示“加载中”或骨架屏。


2. **列表展示：** 展示 Prompt 的 `Title`，可折叠或截断展示部分 `Content` 以供预览。
3. **快捷搜索（P1 优先级）：** 提供顶部搜索框，支持对 `Title` 和 `Content` 进行模糊匹配。
4. **一键填充：** * 鼠标悬浮某一项时有视觉反馈（Hover 态）。
* **点击任意一项，自动将 `Content` 发送至当前活跃网页的输入框内。**


5. **异常引导：**
* 若未配置飞书参数，提示“请先完成配置”，并提供跳转至 Options 页面的按钮。
* 若 API 报错（如无权限、密钥过期），展示易懂的错误提示。



### 4.4 网页注入模块 (Content Script)

**功能描述：** 隐藏在网页背后的脚本，负责将提取的文本安全、准确地填入第三方网站。
**兼容性要求（验收标准）：**

* **适配 ChatGPT (`chatgpt.com`)：** 能够准确定位其动态变化的输入框 DOM 节点。
* **适配 Claude (`claude.ai`)：** 能够处理其富文本 (`contenteditable`) 形式的输入框。
* **注入方式：** 优先采用模拟用户粘贴或触发底层 React/Vue 输入事件的方式，确保输入内容能被目标网站的框架正确识别并激活“发送”按钮。

---

## 5. 非功能需求 (NFR)

### 5.1 安全性

* `App Secret` 属于敏感信息，仅存储在用户本地浏览器中，绝不上传至任何第三方服务器。
* 建议飞书自建应用采用“最小权限原则”，仅授予“多维表：读取”权限，防范数据被意外篡改。

### 5.2 性能与稳定性

* **缓存机制：** 避免每次打开弹窗都消耗飞书 API 额度，合理设置缓存过期时间（如 5 分钟），或提供手动“同步”按钮。
* **容错处理：** 当第三方网站（ChatGPT/Claude）前端页面改版导致 DOM 选择器失效时，插件应抛出友好的报错（“未找到输入框”），而不是静默崩溃。

---

## 6. 后续迭代规划 (Roadmap)

* **v1.1：** 增加“分类筛选”功能（读取多维表的 Category 字段）。
* **v1.2：** 支持参数化 Prompt（如 Prompt 中包含 `[输入你的代码]`，注入后自动高亮或弹出二次输入框）。
* **v2.0：** 反向收藏功能（在 ChatGPT 页面选中文本，右键菜单直接“保存至飞书 Prompt 库”）。
