# 搜索框优化方案对比

## 问题
搜索框占用太多垂直空间，导致可见的 Prompt 列表变少。

## 方案 1：可展开搜索（已实现）✅

### 设计
- 默认隐藏搜索框
- 点击搜索按钮展开
- 显示关闭按钮可收起
- 支持 ESC 键关闭

### 优点
- ✅ 节省空间，列表可见区域更大
- ✅ 界面更简洁
- ✅ 现代化的交互方式
- ✅ 不常用搜索时体验更好

### 缺点
- ⚠️ 需要多一次点击才能搜索
- ⚠️ 搜索功能不够明显

### 适用场景
- Prompt 数量较少（<20 个）
- 用户不经常使用搜索
- 追求简洁的界面

### 效果预览
```
┌─────────────────────────────┐
│ 我的提示词    🔍 ↻ ⚙       │ ← 点击搜索图标
├─────────────────────────────┤
│ [搜索框展开后显示在这里]     │
├─────────────────────────────┤
│ Prompt 1                    │
│ Prompt 2                    │
│ Prompt 3                    │
│ ...                         │
└─────────────────────────────┘
```

## 方案 2：紧凑搜索框

### 设计
- 搜索框始终显示
- 高度减小到 32px
- 内边距减小
- 与标题行更紧凑

### 优点
- ✅ 搜索功能始终可见
- ✅ 无需额外点击
- ✅ 适合频繁搜索

### 缺点
- ⚠️ 仍然占用一定空间
- ⚠️ 界面相对不够简洁

### 实现代码

#### HTML（无需修改）
保持原有结构，但搜索框始终显示。

#### CSS
```css
.search-box {
  padding: 8px 20px;
  border-bottom: 1px solid #f3f4f6;
}

#searchInput {
  height: 32px;
  padding: 0 12px 0 36px;
  font-size: 13px;
}

.search-icon {
  left: 28px;
}
```

### 效果预览
```

│ 我的提示词         ↻ ⚙     │
├─────────────────────────────┤
│ 🔍 [搜索提示词...]          │ ← 更紧凑
├─────────────────────────────┤
│ Prompt 1                    │
│ Prompt 2                    │
│ Prompt 3                    │
│ ...                         │
└─────────────────────────────┘
```

## 方案 3：标题行集成

### 设计
- 搜索框放在标题行
- 标题缩小
- 搜索框宽度自适应

### 优点
- ✅ 最节省空间
- ✅ 搜索始终可见
- ✅ 一行搞定

### 缺点
- ⚠️ 标题和搜索框可能拥挤
- ⚠️ 小屏幕上可能显示不全
- ⚠️ 视觉层次不够清晰

### 实现代码

#### HTML
```html
<header class="header">
  <h1 class="title">提示词</h1>
  <div class="search-compact">
    <i data-lucide="search" class="search-icon"></i>
    <input type="text" id="searchInput" placeholder="搜索..." />
  </div>
  <div class="header-actions">
    <button class="btn-icon btn-refresh" id="refreshBtn">
      <i data-lucide="refresh-cw"></i>
    </button>
    <button class="btn-icon btn-settings" id="settingsBtn">
      <i data-lucide="settings"></i>
    </button>
  </div>
</header>
```

#### CSS
```css
.header {
  display: flex;
  align-items: center;
  gap: 12px;
  padding: 16px 20px;
}

.title {
  font-size: 16px;
  flex-shrink: 0;
}

.search-compact {
  position: relative;
  flex: 1;
  min-width: 120px;
}

.search-compact input {
  width: 100%;
  height: 32px;
  padding: 0 12px 0 32px;
  border: 1px solid #e5e7eb;
  border-radius: 6px;
  font-size: 13px;
}

.search-compact .search-icon {
  position: absolute;
  left: 10px;
  top: 50%;
  transform: translateY(-50%);
  width: 14px;
  height: 14px;
}

.header-actions {
  display: flex;
  gap: 4px;
  flex-shrink: 0;
}
```

### 效果预览
```
┌─────────────────────────────┐
│ 提示词 [🔍 搜索...] ↻ ⚙    │ ← 全在一行
├─────────────────────────────┤
│ Prompt 1                    │
│ Prompt 2                    │
│ Prompt 3                    │
│ ...                         │
└─────────────────────────────┘
```

## 方案 4：浮动搜索

### 设计
- 搜索按钮在右上角
- 点击后浮动层覆盖在列表上方
- 类似移动端的搜索体验

### 优点
- ✅ 完全不占用空间
- ✅ 搜索时全屏体验
- ✅ 最大化列表可见区域

### 缺点
- ⚠️ 实现较复杂
- ⚠️ 可能不符合桌面端习惯

## 推荐方案

### 如果 Prompt 数量少（<20 个）
**推荐：方案 1（可展开搜索）** ✅ 已实现
- 界面最简洁
- 不常用搜索时体验最好
- 现代化交互

### 如果 Prompt 数量多（>20 个）
**推荐：方案 2（紧凑搜索框）**
- 搜索功能始终可见
- 适合频繁搜索
- 实现简单

### 如果追求极致空间利用
**推荐：方案 3（标题行集成）**
- 最节省空间
- 一行搞定所有功能
- 适合宽度足够的场景

## 当前实现（方案 1）

### 特性
- ✅ 默认隐藏搜索框
- ✅ 点击搜索按钮展开
- ✅ 显示关闭按钮
- ✅ 支持 ESC 键关闭
- ✅ 展开动画效果
- ✅ 自动聚焦输入框

### 交互流程
1. 用户点击搜索图标 🔍
2. 搜索框从顶部滑下展开
3. 自动聚焦到输入框
4. 输入关键词实时过滤
5. 点击 ✕ 或按 ESC 关闭搜索
6. 关闭时清空搜索内容

### 键盘快捷键
- `ESC` - 关闭搜索框并清空内容

## 用户反馈建议

可以根据用户使用情况调整：
- 如果用户很少搜索 → 保持方案 1
- 如果用户频繁搜索 → 切换到方案 2 或 3
- 可以添加设置选项让用户自己选择

## 总结

当前实现的**方案 1（可展开搜索）**是一个很好的平衡：
- ✅ 节省空间
- ✅ 界面简洁
- ✅ 交互现代
- ✅ 实现优雅

如果后续发现用户频繁使用搜索，可以考虑切换到方案 2 或 3。
