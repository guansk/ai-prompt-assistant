### 6. 打包上传

#### 打包步骤
1. [ ] 确保所有文件都已更新
2. [ ] 运行 `node build.js` 检查
3. [ ] 创建 ZIP 文件
   ```bash
   # Windows PowerShell
   Compress-Archive -Path manifest.json,popup.html,popup.js,options.html,options.js,content.js,styles,icons -DestinationPath team-prompt-manager.zip -Force
   ```
4. [ ] 验证 ZIP 文件内容
5. [ ] 确保 ZIP 文件小于 50MB

## 📝 注意事项

### 审核时间
- 首次提交：通常 1-3 个工作日
- 更新版本：通常 1 个工作日
- 复杂应用可能需要更长时间

### 常见拒绝原因
1. 权限申请过多或说明不清
2. 隐私政策不完整
3. 功能描述不清晰
4. 截图质量差或误导
5. 违反商店政策

### 避免拒绝的建议
- ✅ 清晰说明每个权限的用途
- ✅ 提供完整的隐私政策
- ✅ 使用高质量的截图
- ✅ 准确描述功能
- ✅ 测试所有功能正常工作
