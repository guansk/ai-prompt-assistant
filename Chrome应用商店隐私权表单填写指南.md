# Chrome 应用商店隐私权表单填写指南

本文档提供 AI Prompt 助手 - 飞书多维表驱动扩展程序在 Chrome 应用商店提交时的隐私权表单标准答案。

---

## 📋 表单填写内容

### 1️⃣ 单一用途说明

**字段名称**：单一用途说明  
**字符限制**：0/1,000

**填写内容**：

```
本扩展程序的单一用途是：帮助用户从飞书多维表中读取和管理 AI Prompt 模板，并在 ChatGPT、Claude、Gemini 等 AI 对话平台上快速填充这些 Prompt 内容。

核心功能包括：
1. 连接飞书开放平台 API，读取用户配置的多维表数据
2. 在扩展弹窗中展示 Prompt 列表，支持搜索和筛选
3. 支持参数化 Prompt 模板，用户可填写变量后生成完整内容
4. 一键将选中的 Prompt 内容自动填充到 AI 平台的输入框中

所有功能均围绕"Prompt 管理与快速调用"这一核心目的，不涉及其他用途。
```

---

### 2️⃣ 权限使用理由

#### storage 权限

**字段名称**：需请求 storage 的理由  
**字符限制**：0/1,000

**填写内容**：

```
storage 权限用于在用户浏览器本地存储以下必要配置信息：

1. 飞书 API 凭证（App ID、App Secret）- 用于访问飞书开放平台 API
2. 飞书多维表配置（App Token、Table ID）- 用于定位用户的 Prompt 数据源
3. Prompt 数据缓存 - 减少 API 调用频率，提升用户体验
4. 用户偏好设置（如 API 域名选择）

所有数据仅存储在用户本地浏览器中，不会上传到任何服务器。用户可随时通过卸载扩展或清除浏览器数据来删除这些信息。此权限是实现扩展核心功能的必要条件。
```

#### activeTab 权限

**字段名称**：需请求 activeTab 的理由  
**字符限制**：0/1,000

**填写内容**：

```
activeTab 权限用于获取当前标签页的基本信息，以便：

1. 判断当前页面是否为支持的 AI 平台（ChatGPT、Claude、Gemini 等）
2. 确定是否显示"自动填充"功能按钮
3. 在用户点击 Prompt 时，向当前活动标签页发送填充指令

此权限仅在用户主动点击扩展图标或 Prompt 项时使用，不会在后台监控或记录用户的浏览行为。该权限是实现"一键填充"核心功能的必要条件。
```

#### scripting 权限

**字段名称**：需请求 scripting 的理由  
**字符限制**：0/1,000

**填写内容**：

```
scripting 权限用于在支持的 AI 平台页面注入内容脚本（content script），以实现 Prompt 内容的自动填充功能。

具体用途：
1. 在 ChatGPT、Claude、Gemini 等 AI 平台页面定位输入框元素
2. 将用户选择的 Prompt 内容填充到输入框中
3. 触发必要的 DOM 事件，确保输入内容被页面正确识别

内容脚本仅在用户主动点击 Prompt 时执行，不会读取、记录或修改页面的其他内容，也不会访问用户的对话历史。此权限是实现扩展单一用途的核心技术要求。
```

#### 主机权限

**字段名称**：需请求主机权限的理由  
**字符限制**：0/1,000

**填写内容**：

```
主机权限用于访问以下两类网站：

【AI 平台】（用于自动填充功能）
- chatgpt.com、chat.openai.com、*.openai.com - ChatGPT 平台
- claude.ai - Claude AI 平台
- gemini.google.com、aistudio.google.com - Google Gemini 平台

【飞书 API】（用于读取数据）
- open.feishu.cn、open.larksuite.com - 飞书开放平台 API 端点

在 AI 平台上，扩展仅在用户点击 Prompt 时将内容填充到输入框，不读取对话内容。
在飞书 API 上，扩展仅读取用户配置的多维表数据，不访问其他飞书资源。

所有网络请求均由用户主动触发，不进行后台数据收集或追踪。
```

### 3️⃣ 远程代码使用

**字段名称**：您正在使用远程代码吗？

**选择**：❌ 不，我并未使用远程代码

**理由字段**：（无需填写，因为选择了"不使用"）

**说明**：本扩展程序的所有代码均包含在扩展包中，不使用任何外部 JavaScript 或 WebAssembly 文件，不使用 eval() 执行动态代码。

---

### 4️⃣ 用户数据收集

**字段名称**：目前或未来您打算向用户收集哪些用户数据？

**选择**：✅ 网站内容

**具体说明**：

扩展程序会访问以下网站内容：

1. **飞书多维表内容**：读取用户配置的多维表中的 Prompt 数据（标题、内容、分类、状态等字段）
2. **AI 平台输入框**：定位并访问 ChatGPT、Claude、Gemini 等平台的输入框 DOM 元素，以便填充 Prompt 内容

**重要说明**：
- ❌ 不收集用户的对话内容或聊天记录
- ❌ 不收集用户的浏览历史
- ❌ 不收集任何个人身份信息
- ✅ 所有配置数据仅存储在用户本地浏览器
- ✅ 不向任何第三方服务器上传数据

---

### 5️⃣ 数据使用披露

**字段名称**：我确认下列披露信息均属实

**必选项**（全部勾选 ✅）：

- ✅ 我不会出于已获批准的用途之外的用途向第三方出售或传输用户数据
- ✅ 我不会为实现与我的产品的单一用途无关的目的而使用或转移用户数据
- ✅ 我不会为确定信用度或实现贷款而使用或转移用户数据

---

### 6️⃣ 隐私权政策网址

**字段名称**：隐私权政策网址  
**字符限制**：0/2,048  
**必填**：是

**填写内容**：

```
https://github.com/[你的用户名]/[你的仓库名]/blob/main/PRIVACY.md
```

**或者**（如果你有独立网站）：

```
https://[你的域名]/privacy
```

**说明**：
- 必须是公开可访问的 HTTPS 链接
- 建议使用 GitHub Pages 或项目 README 中的 PRIVACY.md 文件
- 确保链接长期有效，不会失效

## 📝 注意事项

1. **保持一致性**：表单中的描述必须与 manifest.json 中的权限声明、PRIVACY.md 中的隐私政策保持一致

2. **真实准确**：所有信息必须真实反映扩展的实际功能，虚假信息会导致审核被拒

3. **简洁明了**：使用清晰的语言，避免技术术语过多，让审核人员和用户都能理解

4. **突出重点**：强调"本地存储"、"不上传数据"、"用户主动触发"等隐私保护措施

5. **及时更新**：如果扩展功能有变化，需要同步更新隐私权表单和政策文档
