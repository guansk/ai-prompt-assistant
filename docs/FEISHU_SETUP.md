# 飞书多维表配置指南

## 字段配置

### 必需字段

#### 1. Title（单行文本）
- **字段类型**：单行文本
- **是否必填**：是
- **说明**：Prompt 的标题或简短描述

#### 2. Content（多行文本）
- **字段类型**：多行文本
- **是否必填**：是
- **说明**：完整的 Prompt 内容，将被填充到 AI 输入框

### 可选字段

#### 3. Category（单选）
- **字段类型**：单选
- **是否必填**：否
- **选项配置**：
  - 研发
  - 产品
  - 设计
  - 运营
  - 客服
  - 其他

#### 4. Status（单选）
- **字段类型**：单选
- **是否必填**：否
- **默认值**：启用
- **选项配置**：
  - 启用
  - 停用

## 参数化 Prompt 示例

### 翻译助手

```
请将以下内容从 [源语言] 翻译成 [目标语言]：

专业领域：[?专业领域]

[待翻译内容]
```

### 代码审查

```
请对以下代码进行专业审查：

编程语言：[编程语言]
代码功能：[?代码功能描述]
关注重点：[?需要特别关注的方面]

代码内容：
[代码内容]

请从以下角度进行审查：
1. 代码质量和可读性
2. 潜在的 bug 和安全问题
3. 性能优化建议
4. 最佳实践建议
```

### 会议纪要生成

```
# Role:
你是一位拥有10年经验的专业商务会议秘书。

# Task
请阅读附后的会议转录文本，执行以下操作：
1. 语义修正：修正转录中的错别字，去除口语废话
2. 信息重组：将散落的同一话题内容进行归纳整合
3. 输出纪要：生成精确、详细且逻辑清晰的会议纪要

# Context
* 参会方：[?参会方人员名单]
* 会议主题：[?本次会议的核心主题]
* 术语及备注：[?相关产品术语、背景资料或缩写]

# Output Format
## 1. 会议核心摘要
## 2. 详细议题记录 (按话题分类)
## 3. 核心共识

---

[待转录的文本]
```

## 使用建议

### 标题命名
- 使用简短、描述性的标题
- 便于快速识别和搜索
- 避免过长或模糊的名称

### 内容编写
- 使用清晰的指令语言
- 可以包含占位符（如 [代码]、[需求]）
- 分点列出要求，提高可读性
- 使用参数化语法提升复用性

### 分类管理
- 根据团队实际情况调整分类
- 保持分类体系简洁清晰
- 定期review和优化分类

### 状态控制
- 使用"停用"隐藏过时的 Prompt
- 不要直接删除，保留历史记录
- 定期清理长期停用的内容

### 持续优化
- 根据使用反馈改进 Prompt 质量
- 团队共同维护和优化
- 记录优化历史和原因

## 获取多维表标识

从多维表 URL 中提取信息：

```
https://example.feishu.cn/base/[App Token]?table=[Table ID]&view=xxx
```

- **App Token**：`bascn` 开头的字符串
- **Table ID**：`tbl` 开头的字符串
