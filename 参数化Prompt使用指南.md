# 参数化 Prompt 使用指南 (v1.1)

## 📋 功能概述

v1.1 版本新增了参数化 Prompt 功能，允许在 Prompt 模板中定义变量占位符，用户调用时通过动态表单填写变量值，实现"一个模板，多种场景"的灵活应用。

## 🎯 核心价值

- **提升复用性**：一个 Prompt 模板可以适配多种相似场景
- **减少维护成本**：无需为每个细微差异创建新的 Prompt
- **增强灵活性**：支持必填和选填变量，满足不同粒度的需求
- **保持简洁**：无变量的 Prompt 保持原有的一键填充体验

## 📝 变量语法

### 必填变量
```
[变量名]
```
- 用中括号包裹变量名
- 用户必须填写此字段才能提交
- 表单中显示红色星号 `*` 标识

### 选填变量
```
[?变量名]
```
- 用中括号包裹，变量名前加问号 `?`
- 用户可以留空
- 留空时自动替换为"无"
- 表单中显示灰色"(选填)"提示

## 💡 使用示例

### 示例 1：会议纪要生成器

**飞书多维表配置：**

| 字段 | 内容 |
|------|------|
| Title | 会议纪要生成器 |
| Content | 见下方 |
| Category | 办公 |
| Status | 启用 |

**Content 内容：**
```
# Role:
你是一位拥有10年经验的专业商务会议秘书，擅长从语音转录的文本中提取关键信息、修正逻辑错误，并输出高质量的会议纪要。

# Task
请阅读附后的会议转录文本，执行以下操作：
1. 语义修正：根据上下文和行业背景，智能修正转录中的错别字，去除口语废话。
2. 信息重组：打破原有的流水账结构，将散落在不同时间段的同一话题内容进行归纳整合。
3. 输出纪要：生成一份精确、详细且逻辑清晰的会议纪要。

# Context (业务背景)
* 参会方：[?参会方人员名单]
* 会议主题：[?本次会议的核心主题]
* 术语及备注：[?相关产品术语、背景资料或缩写]

# Output Format
## 1. 会议核心摘要
## 2. 详细议题记录 (按话题分类)
## 3. 核心共识

---

[待转录的文本]
```

**调用效果：**
1. 点击"会议纪要生成器"
2. 弹出表单，包含 4 个输入框：
   - 参会方人员名单 (选填)
   - 本次会议的核心主题 (选填)
   - 相关产品术语、背景资料或缩写 (选填)
   - ⭐ 待转录的文本 (必填)
3. 填写必填项，选填项可留空
4. 点击"确认生成并填入"
5. 完整 Prompt 自动填入 AI 对话框

## 🎨 用户界面

### 表单布局
```
┌─────────────────────────────────┐
│  填写参数                        │
├─────────────────────────────────┤
│                                 │
│  * 必填字段名                    │
│  ┌───────────────────────────┐  │
│  │ 请输入...                  │  │
│  └───────────────────────────┘  │
│                                 │
│  选填字段名 (选填)               │
│  ┌───────────────────────────┐  │
│  │ 选填，可留空               │  │
│  └───────────────────────────┘  │
│                                 │
├─────────────────────────────────┤
│  [返回列表]  [确认生成并填入]    │
└─────────────────────────────────┘
```

### 视觉标识
- **必填项**：红色星号 `*` + 字段名
- **选填项**：字段名 + 灰色"(选填)"
- **错误状态**：未填写必填项时，输入框显示红色边框

## ⚙️ 技术实现

### 变量解析逻辑
```javascript
// 正则表达式匹配 [?变量名] 和 [变量名]
const regex = /\[(\??[^\]]+)\]/g;

// 解析结果
{
  placeholder: "[?参会方]",  // 原始占位符
  name: "参会方",            // 显示名称
  required: false            // 是否必填
}
```

### 变量替换逻辑
```javascript
// 有值：直接替换
finalContent = content.replace("[变量名]", "用户输入的值");

// 选填项为空：替换为"无"
finalContent = content.replace("[?变量名]", "无");
```

## ✅ 最佳实践

### 1. 变量命名
- ✅ 使用清晰、简洁的中文描述
- ✅ 准确描述需要填写的内容
- ❌ 避免使用特殊字符或过长的名称

**好的示例：**
- `[编程语言]`
- `[目标受众]`
- `[?会议主题]`

**不好的示例：**
- `[请在这里输入你想要审查的代码的编程语言名称]` (过长)
- `[lang/language]` (包含特殊字符)

### 2. 必填 vs 选填
- **必填**：核心信息，缺少会导致 Prompt 无法正常工作
- **选填**：补充信息，可以提升效果但非必需

**示例：**
```
翻译任务：
源语言：[源语言]           # 必填，核心信息
目标语言：[目标语言]       # 必填，核心信息
专业领域：[?专业领域]     # 选填，可以提升翻译质量
语气风格：[?语气风格]     # 选填，锦上添花

待翻译内容：
[原文]                    # 必填，核心信息
```

### 3. 选填项的默认值处理
当前实现中，选填项留空时替换为"无"。你也可以根据实际需求调整：

**方案 A：替换为"无"（当前实现）**
```
* 参会方：无
* 会议主题：产品规划讨论
```

**方案 B：删除整行（可选实现）**
```
* 会议主题：产品规划讨论
```

如需修改，可以在 `popup.js` 中调整替换逻辑。

### 4. 表单体验优化
- 合理控制变量数量（建议不超过 5-6 个）
- 将最重要的必填项放在前面
- 为输入框设置合适的行数（短文本 1-2 行，长文本 3-5 行）

## 🔄 向后兼容

无变量的传统 Prompt 保持原有行为：
- 点击后直接填入 AI 对话框
- 不弹出表单
- 用户体验无变化

## 🐛 常见问题

### Q1: 变量名中可以包含空格吗？
A: 可以。`[会议 主题]` 和 `[会议主题]` 都是有效的。

### Q2: 可以在一个 Prompt 中使用多个相同名称的变量吗？
A: 可以。所有同名变量会被替换为相同的值。

### Q3: 如果忘记填写必填项会怎样？
A: 系统会阻止提交，并在输入框显示红色边框提示。

### Q4: 选填项留空后，可以自定义替换内容吗？
A: 当前版本统一替换为"无"。如需自定义，可以修改 `popup.js` 中的替换逻辑。

### Q5: 变量名可以使用英文吗？
A: 可以。`[language]` 和 `[编程语言]` 都是有效的。

## 📊 使用建议

### 适合使用参数化的场景
- ✅ 结构化的 Prompt 模板（如：代码审查、文案润色）
- ✅ 需要根据不同场景调整细节的 Prompt
- ✅ 团队内有多种相似但略有差异的需求

### 不适合使用参数化的场景
- ❌ 完全固定的 Prompt（如：通用的翻译指令）
- ❌ 变量过多（超过 7-8 个）导致填写繁琐
- ❌ 每次使用都需要大量自定义的场景

## 🚀 升级指南

如果你已经在使用 v1.0，升级到 v1.1 后：

1. **现有 Prompt 无需修改**：无变量的 Prompt 保持原有行为
2. **逐步添加变量**：可以选择性地为部分 Prompt 添加变量
3. **测试验证**：建议先在测试环境验证参数化 Prompt 的效果
