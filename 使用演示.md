# 使用演示

## 场景一：研发团队使用代码审查 Prompt

### 1. 在飞书多维表中准备数据

| Title | Content | Category | Status |
|-------|---------|----------|--------|
| Python 代码审查 | 请审查以下 Python 代码，关注：<br>1. 代码质量和可读性<br>2. 潜在的 bug 和安全漏洞<br>3. 性能优化机会<br>4. 是否遵循 PEP 8 规范<br>5. 最佳实践建议 | 研发 | 启用 |

### 2. 在 ChatGPT 中使用

1. 打开 https://chatgpt.com
2. 点击浏览器工具栏的插件图标
3. 在搜索框输入"代码审查"
4. 点击"Python 代码审查"卡片
5. Prompt 自动填充到输入框
6. 粘贴你的代码
7. 发送消息

### 3. 效果

```
请审查以下 Python 代码，关注：
1. 代码质量和可读性
2. 潜在的 bug 和安全漏洞
3. 性能优化机会
4. 是否遵循 PEP 8 规范
5. 最佳实践建议

[你的代码]
```

## 场景五：团队协作 - 持续优化 Prompt

### 1. 初始版本

团队成员 A 创建了一个基础的代码审查 Prompt：

```
请审查以下代码，指出问题。
```

### 2. 团队反馈

- 成员 B：建议增加具体的审查维度
- 成员 C：建议区分不同编程语言
- 成员 D：建议增加输出格式要求

### 3. 优化后版本

在飞书多维表中更新：

```
请审查以下 [语言] 代码，从以下维度进行分析：

1. 代码质量
   - 可读性和可维护性
   - 命名规范
   - 代码结构

2. 功能正确性
   - 逻辑错误
   - 边界情况处理
   - 异常处理

3. 性能优化
   - 时间复杂度
   - 空间复杂度
   - 可优化的点

4. 安全性
   - 潜在的安全漏洞
   - 输入验证
   - 权限控制

5. 最佳实践
   - 是否遵循语言规范
   - 设计模式应用
   - 改进建议

请以表格形式输出，包含：问题描述、严重程度、建议修改。
```

### 4. 效果

- 所有团队成员立即使用最新版本
- 代码审查质量显著提升
- 审查标准统一
- 新人快速学习团队规范

## 高级技巧

### 技巧 1：使用占位符

在 Prompt 中使用占位符，提醒自己补充内容：

```
请分析以下 [产品名称] 的竞品情况：

目标用户：[用户群体]
核心功能：[功能列表]
价格区间：[价格范围]

请从以下维度分析...
```

### 技巧 2：分类管理

使用 Category 字段组织 Prompt：

- 研发：代码审查、重构、测试、文档
- 产品：需求分析、竞品分析、用户研究
- 设计：文案优化、UI 建议、交互设计
- 运营：内容创作、数据分析、活动策划
- 客服：标准话术、问题解答、投诉处理

### 技巧 3：版本管理

利用飞书多维表的历史记录功能：

1. 查看 Prompt 的修改历史
2. 对比不同版本的效果
3. 必要时回滚到旧版本

### 技巧 4：快速搜索

使用关键词快速定位：

- 搜索"Python"找到所有 Python 相关的 Prompt
- 搜索"分析"找到所有分析类的 Prompt
- 搜索"优化"找到所有优化类的 Prompt

### 技巧 5：团队规范

建立团队 Prompt 编写规范：

1. **命名规范**：使用清晰描述性的标题
2. **结构规范**：使用分点列表，便于阅读
3. **更新规范**：重大修改时在团队群通知
4. **审核规范**：新 Prompt 需要团队 Review
