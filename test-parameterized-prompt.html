📝 参数化 Prompt 功能测试指南

功能说明

v1.1 版本新增了参数化 Prompt 功能，支持在 Prompt 中定义变量，用户调用时动态填写。

变量语法

  • 必填变量：[变量名] - 用户必须填写
  • 选填变量：[?变量名] - 用户可以留空

测试示例 1：会议纪要生成器

在飞书多维表中配置以下内容：

Title: 会议纪要生成器

Content:

# Role:
你是一位拥有10年经验的专业商务会议秘书，擅长从语音转录的文本中提取关键信息、修正逻辑错误，并输出高质量的会议纪要。

# Task
请阅读附后的会议转录文本，执行以下操作：
1. 语义修正：根据上下文和行业背景，智能修正转录中的错别字，去除口语废话。
2. 信息重组：打破原有的流水账结构，将散落在不同时间段的同一话题内容进行归纳整合。
3. 输出纪要：生成一份精确、详细且逻辑清晰的会议纪要。

# Context (业务背景)
* 参会方：[?参会方人员名单]
* 会议主题：[?本次会议的核心主题]
* 术语及备注：[?相关产品术语、背景资料或缩写]

# Output Format
## 1. 会议核心摘要
## 2. 详细议题记录 (按话题分类)
## 3. 核心共识

---

[待转录的文本]

测试步骤：

  1. 在飞书多维表中添加上述 Prompt
  2. 打开插件，点击"会议纪要生成器"
  3. 应该弹出表单，包含 4 个输入框：
    • 参会方人员名单 (选填)
    • 本次会议的核心主题 (选填)
    • 相关产品术语、背景资料或缩写 (选填)
    • * 待转录的文本 (必填，带红色星号)
  4. 尝试不填必填项，点击"确认生成并填入"，应该提示错误
  5. 填写必填项后，选填项可以留空或填写
  6. 点击确认后，应该将拼接好的完整 Prompt 填入 AI 对话框

测试示例 2：代码审查模板

在飞书多维表中配置以下内容：

Title: 代码审查模板

Content:

请对以下代码进行专业审查：

编程语言：[编程语言]
代码功能：[?代码功能描述]
关注重点：[?需要特别关注的方面]

代码内容：
[代码内容]

请从以下角度进行审查：
1. 代码质量和可读性
2. 潜在的 bug 和安全问题
3. 性能优化建议
4. 最佳实践建议

预期效果：

  • 必填项：编程语言、代码内容
  • 选填项：代码功能描述、需要特别关注的方面
  • 选填项留空时，替换为"无"

测试示例 3：无变量 Prompt（向后兼容）

在飞书多维表中配置以下内容：

Title: Python 代码重构

Content:

请帮我重构以下 Python 代码，使其更加简洁、高效和符合 PEP 8 规范。

预期效果：

  • 点击后直接填入 AI 对话框，不弹出表单
  • 保持 v1.0 的行为，确保向后兼容

验收标准

  • ✅ 能正确识别必填变量 [变量名]
  • ✅ 能正确识别选填变量 [?变量名]
  • ✅ 必填项未填写时，阻止提交并提示错误
  • ✅ 选填项留空时，替换为"无"
  • ✅ 表单 UI 清晰，必填项有红色星号标识
  • ✅ 选填项有"(选填)"灰色提示
  • ✅ 点击"返回列表"能正常返回
  • ✅ 无变量的 Prompt 保持原有行为（直接填充）
  • ✅ 变量替换后的完整 Prompt 能正确注入到 AI 对话框

注意事项

  • 变量名应该简洁明了，描述清楚需要填写的内容
  • 避免在变量名中使用特殊字符
  • 合理使用必填和选填，提升用户体验
  • 选填项的默认值为"无"，可以根据实际需求调整
