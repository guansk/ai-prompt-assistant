# 发布指南

## 开发模式加载

适合开发和测试：

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择项目根目录

## 打包发布

### 方法一：手动打包

1. 确保所有文件都已准备好：
   ```bash
   node build.js
   ```

2. 创建 ZIP 文件，包含以下文件和目录：
   ```
   manifest.json
   popup.html
   popup.js
   options.html
   options.js
   content.js
   styles/
   icons/
   ```

3. 排除以下文件：
   ```
   .git/
   .vscode/
   node_modules/
   *.md
   build.js
   generate-icons.html
   package.json
   .gitignore
   .gitattributes
   prd.md
   ```

### 方法二：使用命令行

Windows (PowerShell):
```powershell
Compress-Archive -Path manifest.json,popup.html,popup.js,options.html,options.js,content.js,styles,icons -DestinationPath team-prompt-manager.zip -Force
```

macOS/Linux:
```bash
zip -r team-prompt-manager.zip manifest.json popup.html popup.js options.html options.js content.js styles icons
```

## 发布到 Chrome Web Store

### 准备工作

1. 注册 Chrome Web Store 开发者账号
   - 访问：https://chrome.google.com/webstore/devconsole
   - 支付一次性注册费用（$5）

2. 准备宣传素材：
   - 应用图标：128x128 PNG
   - 小型宣传图：440x280 PNG
   - 大型宣传图：1400x560 PNG（可选）
   - 屏幕截图：1280x800 或 640x400 PNG（至少 1 张）

### 发布步骤

1. 登录 Chrome Web Store 开发者控制台

2. 点击"新增项"

3. 上传 ZIP 文件

4. 填写商店信息：
   - **名称**：Team Prompt Manager
   - **简短描述**：团队 Prompt 管理工具，基于飞书多维表实现团队级共享
   - **详细描述**：
     ```
     Team Prompt Manager 是一款轻量级的 Chrome 浏览器插件，帮助团队统一管理和快速使用 AI Prompt。

     核心功能：
     • 一键填充：在 ChatGPT/Claude 页面快速调用团队 Prompt
     • 飞书管理：使用飞书多维表作为数据后台，无需开发管理系统
     • 快速搜索：支持标题和内容的模糊搜索
     • 智能缓存：本地缓存机制，提升响应速度
     • 安全可靠：敏感信息仅存储在本地浏览器

     适用场景：
     • 研发团队共享代码审查/重构模板
     • 产品团队共享需求分析模板
     • 设计团队共享文案/设计提示词
     • 客服团队共享标准化话术

     使用前需要：
     1. 拥有飞书企业账号
     2. 创建飞书自建应用
     3. 创建多维表存储 Prompt 数据

     详细使用说明请访问项目主页。
     ```
   - **类别**：生产力工具
   - **语言**：中文（简体）

5. 上传宣传素材

6. 设置隐私政策：
   ```
   本插件不收集任何用户数据。
   所有配置信息（飞书 API 凭证）仅存储在用户本地浏览器中。
   插件仅在用户主动操作时访问飞书 API 读取多维表数据。
   ```

7. 选择可见性：
   - 公开：所有人可见
   - 不公开：仅通过链接访问
   - 私有：仅指定用户可见

8. 提交审核

### 审核时间

- 通常需要 1-3 个工作日
- 首次提交可能需要更长时间
- 审核通过后会收到邮件通知

## 企业内部分发

如果不想公开发布，可以通过以下方式在企业内部分发：

### 方法一：共享 CRX 文件

1. 在开发者模式下打包扩展程序
2. 生成 .crx 文件和 .pem 密钥文件
3. 将 .crx 文件分发给团队成员
4. 团队成员拖拽到 chrome://extensions/ 安装

注意：Chrome 可能会提示"无法从该网站添加应用"，需要开启开发者模式。

### 方法二：企业策略部署

适合大型企业：

1. 使用 Google Workspace 管理控制台
2. 上传扩展程序到企业应用商店
3. 通过策略自动部署到员工浏览器

### 方法三：源码分发

1. 将项目代码托管到内部 Git 仓库
2. 团队成员克隆代码
3. 以开发者模式加载

## 版本更新

### 更新版本号

编辑 `manifest.json`：
```json
{
  "version": "1.1.0"
}
```

版本号规则：
- 主版本号：重大功能变更
- 次版本号：新增功能
- 修订号：Bug 修复

### 发布更新

1. 修改代码
2. 更新版本号
3. 重新打包
4. 上传到 Chrome Web Store
5. 用户会自动收到更新

## 常见问题

### Q: 审核被拒怎么办？

A: 查看拒绝原因，常见问题：
- 权限申请过多
- 隐私政策不完整
- 功能描述不清晰
- 违反商店政策

### Q: 如何撤回已发布的版本？

A: 在开发者控制台可以：
- 取消发布（下架）
- 删除特定版本
- 回滚到旧版本

### Q: 可以收费吗？

A: Chrome Web Store 支持：
- 一次性付费
- 订阅制
- 应用内购买

需要设置 Google Payments 商家账号。
