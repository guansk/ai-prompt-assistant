# 参数化 Prompt 缓存功能说明

## 功能特性

### 1. 实时自动保存
- 用户在表单中输入参数时，每次输入都会自动保存到本地缓存
- 使用 `input` 事件监听，确保每个字符的输入都被保存
- 缓存键格式：`paramCache_${promptId}`

### 2. 表单状态持久化
- 当用户打开参数填写表单时，会记录当前正在编辑的 prompt ID
- 即使用户切换到其他标签页或关闭 popup，再次打开时会自动恢复表单
- 使用 `currentFormPromptId` 记录当前表单状态

### 3. 自动恢复机制
- Popup 初始化时检查是否有未完成的表单
- 如果有，自动加载对应的 prompt 并恢复表单（不显示列表）
- 从缓存中读取之前填写的参数值并自动填充
- 如果没有未完成的表单，正常显示 prompt 列表

### 4. 清除缓存功能
- 表单头部提供"清除缓存"按钮
- 点击后会删除该 prompt 的所有缓存参数值
- 同时清空表单中的所有输入框

## 使用流程

1. 用户点击带参数的 prompt
2. 打开参数填写表单
3. 开始填写参数（自动实时保存）
4. 如果需要切换标签页或关闭 popup：
   - 填写的内容已自动保存
   - 表单状态已记录
   - 再次打开 popup 会直接显示表单（不是列表）
   - 所有填写内容自动恢复
5. 填写完成后点击"确认生成并填入"
6. 参数值被永久保存，下次使用该 prompt 时自动填充
7. 点击"返回列表"或"刷新"按钮会清除表单状态记录

## 技术实现

### 缓存存储
```javascript
// 缓存键
const cacheKey = `paramCache_${prompt.id}`;

// 保存格式
{
  "paramCache_xxx": {
    "变量名1": "用户输入的值1",
    "变量名2": "用户输入的值2"
  },
  "currentFormPromptId": "xxx"
}
```

### 实时保存
```javascript
textarea.addEventListener('input', async (e) => {
  const currentCache = await chrome.storage.local.get([cacheKey]);
  const currentValues = currentCache[cacheKey] || {};
  currentValues[variable.name] = e.target.value.trim();
  await chrome.storage.local.set({ [cacheKey]: currentValues });
});
```

### 状态恢复
```javascript
// 记录当前表单
await chrome.storage.local.set({ currentFormPromptId: prompt.id });

// 初始化时恢复表单
const formState = await chrome.storage.local.get(['currentFormPromptId']);
if (formState.currentFormPromptId && allPrompts.length > 0) {
  const prompt = allPrompts.find(p => p.id === formState.currentFormPromptId);
  if (prompt) {
    await showVariableForm(prompt, parseVariables(prompt.content));
    return; // 显示表单，不显示列表
  }
}
// 没有未完成的表单，显示列表
renderPrompts();
```

### 清除表单状态
```javascript
// 取消按钮
await chrome.storage.local.remove('currentFormPromptId');

// 确认填充后
await chrome.storage.local.remove('currentFormPromptId');

// 刷新按钮
await chrome.storage.local.remove('currentFormPromptId');
```

## 优势

1. **防止数据丢失**：即使意外关闭 popup 也不会丢失填写进度
2. **无缝恢复**：再次打开 popup 直接显示表单，不需要重新找 prompt
3. **提升用户体验**：不需要重复填写相同的参数
4. **智能恢复**：自动识别并恢复未完成的表单
5. **灵活控制**：用户可以随时清除缓存重新填写

## 注意事项

- 缓存数据存储在 Chrome 本地存储中，不会同步到云端
- 每个 prompt 的缓存是独立的，互不影响
- 清除浏览器数据时会同时清除缓存
- 点击"返回列表"、"刷新"或"确认填充"会清除表单状态记录
- 下次打开 popup 会正常显示列表，除非有新的未完成表单
